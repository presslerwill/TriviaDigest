2. GitHub Actions CI/CD

3. Question deduplication — prompt injection (Issue #28)

4. AI Eval Harness ← biggest differentiator
Add a generation_runs table that logs every time Claude generates questions: timestamp, model version, prompt version, latency (ms), estimated token cost, per-question category and difficulty distribution. Then build a simple admin page (/admin) that surfaces this as a dashboard.

5. Prompt versioning
Add a prompt_versions table. Each generation run references a prompt version ID. This makes the iteration history visible — a concrete "prompt engineering" artifact, not just a claim.

Tier 3 — Adds Python ecosystem + RAG (2-4 weeks)
6. Python FastAPI service for question generation
Right now the generator is a Deno Edge Function (TypeScript). Rewrite it as a Python FastAPI service you containerize with Docker. This adds Python + Docker to your resume and you can run it on AWS Lambda (via a container image) or ECS. The AI logic stays the same — you're just changing the runtime.

7. RAG step in question generation + pgvector deduplication
Two complementary features sharing the same infrastructure:

  RAG grounding: Before calling Claude, fetch a few relevant facts from a small curated knowledge base (even PostgreSQL full-text search over a facts table works). Pass them as context in the prompt. Reduces hallucination; resume framing: "retrieval-augmented generation."

  pgvector semantic deduplication: Enable the pgvector Supabase extension. When a question is inserted, generate and store an embedding. At generation time, run a cosine similarity search — reject any candidate with similarity > 0.85 to an existing question. Scales indefinitely. Resume framing: "vector similarity search," "embedding pipelines," "semantic deduplication." Replaces the prompt-injection approach from #3 once question volume makes that impractical.

Tier 4 — AWS migration (1-2 months, highest effort, hits the most postings)
8. Deploy to AWS
Migrate from Vercel/Supabase to AWS: Next.js on Lambda@Edge or App Runner, PostgreSQL on RDS, question-generation service on Lambda (container). Provision it with AWS CDK (TypeScript CDK is fine — matches your existing stack). This is the biggest lift but every single posting in your target list mentions AWS explicitly.

9. WebAuthn/passkey auth
Wire up actual passkey login using your Ping Identity background. Replace localStorage replay prevention with real auth. This is your existing differentiator and relatively cheap to add since Supabase Auth is already configured in the project.
